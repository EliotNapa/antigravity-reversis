# リバーシWebアプリ 実装計画

静的なWebサイトとして動作する、モダンで高機能なリバーシ（オセロ）のWebアプリを構築します。
本アプリは人間vsCPUの対戦をサポートし、CPUは5段階の強さを選択可能です。Web Workerを活用して、高度なCPU思考中でもブラウザがフリーズしない快適なユーザー体験（UX）を提供します。

## UI・デザイン方針

Web Application Developmentのガイドラインに従い、ユーザーを惹きつけるプレミアムなデザインを構築します。

- **テーマ**: ダークモードを基調としたネオン/サイバーパンク風のモダンデザイン、または洗練されたグラスモルフィズム。
- **盤面**: 3D的な奥行き感のあるデザイン。石が裏返る際は、CSSの3D Transformsを用いたスムーズなフリップアニメーションを実装。
- **タイポグラフィ**: Google Fontsの `Outfit` または `Inter` をインポートして使用。
- **レスポンシブ**: スマートフォン、タブレット、PCのすべての画面サイズで盤面が適切にフィットするレスポンシブレイアウト。
- **インタラクション**: 
  - 石を置ける場所（合法手）のハイライト表示。
  - CPUの思考中を示すインジケータ（プログレスバーやアニメーション）。
  - ゲーム終了時のアニメーションと勝敗リザルト画面。
  - Web Audio APIによる、石を置いた時のSE（効果音）の生成（外部音声ファイル不要）。

## CPUの強さ設計（5段階）

CPUの探索ロジックはブラウザのメインスレッドをブロックしないよう、**Web Worker**上で非同期に実行します。これにより、高難易度（レベル5）での深い探索中もUIアニメーションが滑らかに動き続けます。

| レベル | 難易度 | アルゴリズム・特徴 | 探索の深さ |
| :--- | :--- | :--- | :--- |
| **レベル1** | 超初級 | **完全ランダム**<br>打てる場所（合法手）の中からランダムに選択します。 | - |
| **レベル2** | 初級 | **静的評価値（即時評価）**<br>盤面評価テーブルに基づき、現時点で最も評価値が高い場所を選択します（角の優先など）。 | 深さ 1 |
| **レベル3** | 中級 | **ミニマックス法 (Minimax)**<br>評価テーブルを使用し、相手が最善手を打ってくることを前提に数手先を予測します。 | 深さ 3 |
| **レベル4** | 上級 | **アルファ・ベータ法 (Alpha-Beta)**<br>探索の無駄な枝を刈り取るAlpha-Beta枝刈りを導入し、より深く先読みします。 | 深さ 5 |
| **レベル5** | 最強 | **Alpha-Beta + 動的評価 + 終盤完全読み切り**<br>盤面の静的評価に加え、「石の確定度」「着手可能数（開放度）」を組み合わせた高度な評価関数を使用。終盤（残り10手以下）は終盤専用ソルバーにより勝敗を完全読み切りします。 | 深さ 7<br>(終盤は完全読み切り) |

### 盤面評価値テーブルの設計（レベル2〜5で共通利用）
盤面の各マスの重要度（角は高く、角の隣（X打ち、C打ち）は相手に角を取られやすいため低い）を数値化します。
```
  120, -20,  20,   5,   5,  20, -20, 120,
  -20, -40,  -5,  -5,  -5,  -5, -40, -20,
   20,  -5,  15,   3,   3,  15,  -5,  20,
    5,  -5,   3,   3,   3,   3,  -5,   5,
    5,  -5,   3,   3,   3,   3,  -5,   5,
   20,  -5,  15,   3,   3,  15,  -5,  20,
  -20, -40,  -5,  -5,  -5,  -5, -40, -20,
  120, -20,  20,   5,   5,  20, -20, 120
```

```

├── css/
│   └── style.css                   <- スタイルシート（グラスモルフィズム、アニメーション）
└── js/
    ├── reversi.js                  <- リバーシのゲームロジッククラス（盤面管理、ルール判定）

    ├── sound.js                    <- Web Audio APIによる効果音ジェネレータ
    └── app.js                      <- UI制御、ゲーム進行管理、Workerとの通信
```

## 各ファイルの役割と実装予定内容

### 1. `index.html`
- ゲーム全体のレイアウト（タイトル、難易度選択、手番表示、石の数カウント、盤面、コントロールボタン）。
- 難易度設定（1〜5段階）のドロップダウンまたはセグメントコントロール。
- ゲーム開始/リセット、プレイヤーの手番（先手・後手）選択。

### 2. `css/style.css`
- ダーク系の美しいカラーパレット（ダークグレー、ネオンブルー、ネオングリーン）。
- 盤面と石のスタイリング。石はグラデーションやシャドウを用いて立体感を表現。
- 石の反転アニメーション（`.flip` クラス付与時に CSS `transform: rotateY(180deg)` で回転）。
- CPU思考中のローディングアニメーション。

### 3. `js/reversi.js`
- `Board`クラス:
  - 8x8 of Board（0: 空き, 1: 黒/先手, -1: 白/後手）。
  - 指定マスに石を置けるかの判定（合法手判定）。
  - 石を置いて反転させる処理。
  - パス判定、ゲーム終了判定。
  - 現在の石の個数のカウント。

### 4. `js/cpu-worker.js` (Web Worker)
- メインスレッドから現在の盤面とCPUの難易度レベル、CPUの色を受け取る。
- 難易度に応じたアルゴリズムを実行：
  - **レベル1**: ランダム探索。
  - **レベル2**: 深さ1の評価値最大化。
  - **レベル3**: 深さ3のMinimax。
  - **レベル4**: 深さ5のAlpha-Beta。
  - **レベル5**: 深さ7のAlpha-Beta ＋ 動的評価 ＋ 終盤完全読み切り。
- 最善手（X, Y座標）をメインスレッドに返却。

### 5. `js/sound.js`
- Web Audio APIを利用したビープ音/エフェクト音の合成。
- 石を置いた時の「コトッ」という音、パス時の警告音、勝利/敗北時のファンファーレを生成。

### 6. `js/app.js`
- 画面のDOM操作、イベントリスナー登録。
- ゲームのメインループ（プレイヤーのターン -> 石を置く -> CPUのターン -> Worker呼び出し -> 結果反映 -> プレイヤーのターン...）。
- CPUの思考中に「思考中...」の進捗アニメーションを表示。
- 音声のオン/オフ切り替え機能。

## 検証計画

### 自動テスト / ユニットテスト
- ゲームロジック（`reversi.js`）の検証用スクリプトを `scratch/` ディレクトリに用意し、Node.js等で実行してテストします。
  - 石の反転が正しく行われるか。
  - パスが発生した時に正しく手番が移るか。
  - ゲーム終了時に正しく勝敗が決まるか。

### 手動検証
- 実際にブラウザでアプリを起動し、以下の観点をテストします。
  - CPUレベル1〜5で正常に動作し、フリーズしないこと。
  - 石が置かれた際のアニメーションや効果音が正しく動作すること。
  - レレスポンシブデザインにより、PC・スマホ双方で崩れないこと。
